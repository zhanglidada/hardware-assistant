## 优化计划

### 现状分析
- 现有性能排行页面包含 CPU、GPU、手机三个主标签
- CPU 标签下有三个子选项：全部、桌面CPU、移动CPU
- 当前使用胶囊形状设计，选中状态为浅蓝色背景和蓝色文字
- 用户要求改为卡片式设计

### 优化方案

#### 1. 样式优化
- **选中状态**：
  - 背景色使用蓝色 (#007aff)
  - 文字颜色使用白色 (#ffffff)
  - 卡片式矩形设计，轻微圆角 (8rpx)
  - 增加轻微的阴影效果，提升层次感

- **未选中状态**：
  - 背景色使用白色 (#ffffff)
  - 文字颜色使用灰色 (#666666)
  - 白色卡片设计，带有浅灰色边框
  - 与选中状态形成明显对比

- **过渡动画**：
  - 增加 0.3s 的平滑过渡效果
  - 点击时添加轻微的缩放反馈

#### 2. 代码修改
- **修改 `src/pages/ranking/index.vue` 文件**：
  - 更新 `.sub-tab-item` 类的样式，改为卡片式设计
  - 优化 `.sub-tab-item.active` 类的样式
  - 保持现有的 HTML 结构和逻辑不变

#### 3. 实现效果
- 点击子选项时，当前选项变为蓝色背景白色文字的卡片
- 其他选项保持白色背景灰色文字的卡片
- 动画过渡流畅，符合 iOS 18 设计风格
- 提升用户体验，使选中状态更加清晰明了

### 技术实现
- 使用现有的 Vue 3 Composition API 结构
- 保持 TypeScript 类型安全
- 仅修改 SCSS 样式部分，不涉及逻辑变更
- 确保在微信小程序环境中正常显示
